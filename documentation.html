Configuration

Environment Variables

# Server Configuration
FLASK_PORT=5000           # Web server port
FLASK_ENV=production      # Environment mode

# DuraCloud Settings
HOST=                    # Your institution's DuraCloud host (required)

# File Paths
DOWNLOAD_DIR=            # Download directory path
CONTENT_LIST_DIR=        # Content list directory
LOG_DIR=                 # Log file directory

Docker Configuration

version: '3.8'
services:
  duracloud-retrieval:
    build:
      context: ../..
      dockerfile: app/docker/Dockerfile
    ports:
      - "${FLASK_PORT:-5000}:5000"
    volumes:
      - ../../downloads:/app/downloads
      - ../../content:/app/content
      - ../../duracloud-logs:/app/logs
    env_file:
      - ../../app/.env

Troubleshooting Guide

Unable to connect to DuraCloud

  • Check network connectivity
  • Verify DuraCloud host URL
  • Confirm credentials
  • Check firewall settings
  • Verify proxy configuration

Downloads failing or incomplete

  • Check available disk space
  • Verify file permissions
  • Monitor network stability
  • Check Java heap space
  • Review concurrent download limits

Container not starting

  • Check for port conflicts
  • Verify volume permissions
  • Review environment variables
  • Examine Docker logs
  • Monitor resource limits

Update Procedures

Application Updates

  1. Stop the application
    cd app/docker
docker compose down
  2. Update code
    # Clone the repository if you haven't already
git clone https://github.com/kgitbit/duracloud-retrieval-web.git
cd duracloud-retrieval-web

# Or update existing repository
git pull origin main
  3. Rebuild and restart
    docker compose up --build -d

Driver Updates

  1. Download new driver version
  2. Update configuration
  3. Test in staging environment
  4. Deploy to production
  5. Verify functionality
